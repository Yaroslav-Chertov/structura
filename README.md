# Structura — SaaS-планер продуктивности

> Коммерческий SaaS-лендинг с воронкой конверсии: захват лида → письмо с оплатой → выдача доступа. Next.js 16, React 19, TypeScript, SCSS.

**Продакшн:** [structuraplaner.ru](https://www.structuraplaner.ru)

---

## О проекте

Structura — цифровой планер продуктивности, распространяемый как платный продукт. Проект закрывает полный коммерческий цикл — от первого визита до получения оплаты и выдачи доступа — с минимальной инфраструктурой.

Основная инженерная задача: выстроить рабочую воронку продаж без отдельного бэкенд-сервера, при этом удержав Lighthouse 95+ на конверсионном лендинге.

## Архитектура

```
app/
├── api/
│   └── contact/        # Захват лида → уведомление через Resend
├── components/         # Компонентная структура по фичам
├── sections/           # Секции страницы (Hero, Benefits, CTA и др.)
├── styles/             # Глобальные SCSS + модули по компонентам
└── page.tsx            # SSR точка входа
```

**Ключевые решения:**

- **SSR по умолчанию** — каждая секция рендерится на сервере. Никаких спиннеров загрузки, layout shift отсутствует, чистый HTML для краулеров.
- **Без отдельного бэкенда** — логика захвата лида и отправки писем живёт внутри Next.js API Routes. Меньше инфраструктуры, проще деплой.
- **Serverless-совместимость** — stateless обработчики, без постоянных соединений. Деплоится на Vercel или любой Node.js хост без дополнительной конфигурации.
- **SCSS Modules** — стили изолированы на уровне компонента. Нет коллизий классов, нет загрязнения глобального пространства имён.

## Воронка конверсии

```
Лендинг
    ↓
Форма захвата (email / Telegram)
    ↓
API Route → Resend (уведомление владельцу о новом лиде)
    ↓
Владелец отправляет пользователю ссылку на оплату ЮKassa
    ↓
Пользователь оплачивает → владелец отправляет доступ к планеру
```

Текущая реализация намеренно простая — минимум технической сложности на старте, фокус на проверке спроса. Автоматизация оплаты и выдачи доступа запланирована в следующей итерации.

## Производительность

| Метрика                | Результат                                      |
| ---------------------- | ---------------------------------------------- |
| Lighthouse Performance | 95+                                            |
| First Contentful Paint | < 1.2s                                         |
| Стратегия рендеринга   | SSR                                            |
| JS-бандл               | Минимизирован (без клиентских запросов данных) |

Применённые оптимизации:

- Статические ассеты раздаются с cache-заголовками
- Изображения оптимизированы через `next/image`
- `"use client"` используется только там где нужна интерактивность
- Swiper загружается только для секции карусели (code splitting)

## Локальный запуск

```bash
# Установка зависимостей
npm install

# Настройка окружения
cp .env.example .env.local
# Заполнить: RESEND_API_KEY

# Dev-сервер
npm run dev

# Продакшн-сборка
npm run build
npm run start
```
